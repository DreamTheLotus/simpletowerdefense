### 有关运行时线程安全问题
不加线程锁的原因：
- js本身的单线程特性保证无论是多少条假线程，最终都只在一条线程运行
- 由于炮台是遍历怪物集合的，即便，可能存在炮台在锁定怪物时怪物已经离开了炮台范围区，那就当个特性不管就行了，反正别的游戏都是这情况的(doge)
- 会不会怪物死亡了，但炮台在扫描时发现死亡的怪物？
	- 不会发生，因为在炮台线程完成一轮攻击任务之后才会有可能轮到怪物行动线程的运行
- 暂时就这么多了
